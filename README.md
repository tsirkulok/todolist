🚀 Task Management System (Backend API)Це потужний та безпечний API для управління проектами та завданнями. Система реалізує рольову модель доступу (RBAC), дозволяючи адміністраторам керувати робочим процесом, а клієнтам — відстежувати прогрес лише своїх проектів.🌟 Ключові особливостіНадійна авторизація: Використання JWT (JSON Web Tokens) для захисту ендпоінтів.Безпека даних: Хешування паролів за допомогою bcrypt.Рольова модель (RBAC): Чітке розмежування прав між admin та client.Реляційна база даних: Оптимізовані SQL-запити з використанням JOIN для забезпечення приватності клієнтських даних.Валідація статусів: Контроль життєвого циклу завдань (todo -> inprogress -> completed).🛠 Технологічний стекRuntime: Node.jsFramework: Express.jsDatabase: MySQL / MariaDBLibraries: mysql2, jsonwebtoken, bcrypt, dotenv🏗 Схема бази данихСистема базується на трьох взаємопов'язаних таблицях:users: Зберігає облікові дані та ролі.projects: Належать конкретним клієнтам (client_id).tasks: Завдання, що входять до проектів (project_id).🔑 Доступ та безпекаРольПрава доступуAdminПовний CRUD: створення, редагування, зміна статусів та видалення будь-яких завдань.ClientRead-Only: перегляд завдань лише тих проектів, де користувач вказаний як власник.🚀 Швидкий запуск1. Налаштування середовищаСтворіть файл .env у корені проекту:Фрагмент кодуDB_HOST=localhost
DB_USER=root
DB_PASS=ваш_пароль
DB_NAME=project_tracker
JWT_SECRET=ваш_секретний_ключ
PORT=3000
2. Встановлення та запуск Bash npm install
npm start
📡 API Ендпоінти🔐 АвторизаціяPOST /auth/register — реєстрація нового клієнта.POST /auth/login — отримання токена доступу.📋 Завдання (Потрібен Bearer Token)GET /tasks/readAll — список завдань (автоматична фільтрація за власником).POST /tasks/add — створення завдання (Admin only).PATCH /tasks/update/:id — зміна тексту завдання (Admin only).PATCH /tasks/update-status/:id — оновлення прогресу (Admin only).DELETE /tasks/delete/:id — видалення завдання (Admin only).
