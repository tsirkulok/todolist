# 🚀 Task Management System (Backend API)

Це професійний REST API для управління проектами та завданнями. Система реалізує рольову модель доступу (RBAC), що дозволяє адміністраторам повністю керувати процесом, а клієнтам — переглядати лише власні завдання.

## 🌟 Основні можливості

* **JWT Авторизація:** Захищений доступ до даних за допомогою JSON Web Tokens.
* **Рольова модель (RBAC):** Чітке розмежування прав між `admin` та `client`.
* **Безпека:** Хешування паролів за допомогою бібліотеки `bcrypt`.
* **Реляційна база даних:** Використання складних SQL-запитів (`JOIN`) для гарантування приватності даних.
* **Валідація статусів:** Контроль життєвого циклу завдань (`todo`, `inprogress`, `completed`).

---

## 🛠 Технологічний стек

* **Runtime:** Node.js
* **Framework:** Express.js
* **Database:** MySQL
* **Libraries:** `mysql2`, `jsonwebtoken`, `bcrypt`, `dotenv`

---

## 🏗 Структура бази даних

Проект використовує реляційну схему з трьома основними таблицями:

1.  **`users`**: Облікові записи користувачів та їхні ролі.
2.  **`projects`**: Проекти, що належать конкретним клієнтам (`client_id`).
3.  **`tasks`**: Конкретні завдання, прив'язані до проектів (`project_id`).

---

## 🔑 Права доступу

| Роль | Доступні дії |
| :--- | :--- |
| **Admin** | Створення, редагування, видалення завдань та зміна статусів. |
| **Client** | **Тільки читання (Read-Only)**: перегляд завдань лише своїх проектів. |

---

## 🚀 Як запустити проект

### 1. Налаштування бази даних
Створіть базу даних `project_tracker` та виконайте SQL-скрипт для створення таблиць (див. файл зі схемою).

### 2. Налаштування середовища
Створіть файл `.env` у корені проекту та заповніть його:
```env
DB_HOST=localhost
DB_USER=root
DB_PASS=ваш_пароль
DB_NAME=project_tracker
JWT_SECRET=ваш_секретний_ключ
PORT=3000
